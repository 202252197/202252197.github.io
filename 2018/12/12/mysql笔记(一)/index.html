数据库的好处

1.可以持久化数据到本地,一般存放在C盘中的ProgramData的mysql文件里
2.结构化查询

数据库的常见概念

1.DB：数据库,存储数据容器
2.DBMS:数据库管理系统,又称之为数据库软件或数据库产品,用于创建或管理DB
3.SQL：结构化语言,用于和数据库通信的语言,不是某个数据库软件特有的,而是几乎所有的主流数据库软件统用的语言

数据库存储数据的特点

1.数据存放到表中,然后表再放到库中
2.一个库中可以有多张表,每张表具有唯一的表名用来标识自己
3.表中有一个或多个列,列又称之为"字段"，相当于java中"属性"
4.表中的每一行数据,相当于java中"对象"

SQL简介

什么是SQL

Structured Query Language简称SQL,结构化查询语言,数据库管理系统通过SQL语言来管理数据库中的数据

SQL语言的组成部分

DDL(Data Defination Language):数据定义语言,主要用于定义数据库、表、视图、索引和触发器等。像DROP,CREATE,ALTER等语句 
DML(Data Manipulation Language):主要包括对数据的增删改。INSERT插入数据、UPDATE更新数据、DELETE删除数据。
DQL(Data Query Language):数据检索语句,用来从表中获得数据,确定数据怎样在应用程序中给出,像SELECT查询数据。
DCL(Data Control Language):数据控制语言,主要用于控制用户的访问权限。像GRANT, REVOKE, COMMIT, ROLLBACK等语句。

注意事项

1.1064错误:代表语法错语仔细检查。
2.Truncate和delete是有区别的,在于truncate相当于删表再重建一张同样结构的表,操作后得到一张全新表,而delete是从删除所有的层面来操作的。Truncate相当于把学籍表扔了从画一张,而delete相当于用橡皮把数据擦掉了。全删的话使用Truncate删除更快。
3.1264错误:代表设置的值超出了此类型的范围。
4.1267错误:排序规则不一致,就是两个表的字段排序规则不一样。

往哪张表添加行 表名==insert into lvshihao

1.不指定具体的字段名
    INSERT [INTO] 表名 VALUES|VALUE(值....);
2.列出指定字段
    INSERT [INTO] 表名(字段名称1,...) VALUES|VALUE(值1,...);

修改表的语法

一张表创建完毕,有了N列之后还有可能要增加或删除或修改列
添加字段
    Alter table 表名 add 列名称 列类型 列参数 
    例如:Alter table lvshihao add name varchar(255);
    Alter table 表名 add 列名称 列类型 列参数 after 指定列[后面]
    例如:Alter table lvshihao add age int(10) after password;
    Alter table 表名 add 列名称 列类型 列参数 first 添加到首位
    例如:Alter table lvshihao add age int(10) first;
    可以一次性添加多个如
    Alter table lvshihao add name varchar(255),add age int;
删除字段
    Alter table 表名 drop 列名称
    例如:Alter table lvshihao drop name;
    可以一次性删除多个如
    Alter table lvshihao drop name,drop age;
修改字段
    Alter table 表名 modify 列名称 新类型 新参数 [FIRST|AFTER 字段名称]（不能修改列名称）
    例如:Alter table lvshihao modify name varchar(455);
修改字段名称
    Alter table 表名 change 旧列名称 新列名 新类型 新参数 [FIRST|AFTER 字段名称]
    例如:Alter table lvshihao change name names int;
添加默认值
    Alter table 表名 Alter 字段名 SET DEFAULT 默认值;
删除默认值
    alter table 表名 alter 字段名 drop default;
添加主键
    alter table 表名 add [constraint [symbol]] primary key [index_type](字段名称,..);
    alter table 表名 add primary key(id,card);
    alter table 表名 add constraint symbol primary key index_type(id);
删除主键
    alter table 表名 drop primary key;
    (注意:在删除一个字段主键的时候,如果此字段是自增长的话应该首先去掉自增长)
    去掉自增长可以使用,下面虽然不能去掉主键,但是可以去掉主键以外的属性
    alter table 表名 modify id int unsigned;
添加唯一
    alter table 表名 add [constraint [symbol]] unique [index|key] [索引名称] (字段名称,...) 
删除唯一
    alter table 表名 drop {index|key} 索引名称
修改表的存储引擎
    alter table 表名 engine=存储引擎名称
设置自增长的值
    alter table 表名 auto_increment=值(下次插入值,只要不指定自增字段的值,那么下次就是100)
删除外键
    ALTER TABLE employee drop foreign key emp_fk_dep;
添加外键
    ALTER TABLE employee add constraint emp_fk_dep foreign key(depId) references department(id);

3.每条命令根据需要,可以进行缩进或换行

sql模糊查询-like关键字

% 通配任意个字符
_ 通配单一字符

select * FROM blogbbs WHERE TITLE LIKE '_吕世昊_';    如:数据为"爱吕世昊心"
NOT LIKE (查找关键字中没有吕世昊的数据)
    select * FROM blogbbs WHERE TITLE NOT LIKE '%吕世昊%';

sql分组与过滤结果集

分组GROUP BY
    SELECT * from blogbbs GROUP BY LABEL;
按照多个字段分组
    SELECT * FROM blogbbs GROUP BY LABEL,NAME;
WITH ROLLUP 分组之后再次统计
    select COUNT(*),sum(username),avg(username),sex from cms_user where sex is not nul
l group by sex WITH ROLLUP;
HAVING 过滤结果集(通过HAVING子句对分组结果进行二次筛选)
    SELECT * from blogbbs GROUP BY LABEL HAVING ID>50;

    SELECT * FROM 表名 ORDER BY RAND();

    例如:SELECT TIME FROM lvshihao ORDER BY TIME ASC;
从大到小（DESC）
    例如:SELECT TIME FROM lvshihao ORDER BY TIME DESC;
order by 子句中可以支持单个字段,多个字段,表达式,函数,别名

连接查询-笛卡尔集

select gieName,boyName from beauty,boys
笛卡尔集的错误情况
select count(*) from beauty 假设输出12行
select count(*) from boys 假设输出4行
最终结果:12*4=48行
发生原因:没有有效的连接条件
如何避免:添加有效的连接条件

连接查询的分类

sql92支持（内连接）
sql99支持所有连接
内连接：等值连接，非等值连接,自连接
外连接：左外连接，右外连接,全外连接
交叉连接

等值连接

select gieName,boyName from beauty,boys WHERE beauty.id=boys.userid (sql92语法)
select gieName,boyName from beauty INNER JOIN boys ON beauty.id=boys.userid (sql99语法)
意思为:第一个表中的id如果等于第二个表中的userid的话就返回结果
1.多表等值连接的结果为多表的交集部分
2.n表示连接,至少需要n-1个连接条件
3.多表的顺序没有要求
4.一般需要为表起别名

非等值连接

select * from beauty,boys WHERE beauty.money BETWEEN boys.moeny AND boys.moneys;  (sql92语法)
select * from beauty INNER JOIN boys ON beauty.money BETWEEN boys.moeny AND boys.moneys; (sql99语法)
意思为:非等值连接就是不是直接等于而是进行判断返回来的结果

自连接

SELECT b.PREVIEW,b.COMMENT,m.PREVIEW,m.COMMENT FROM blogbbs b,blogbbs m WHERE  b.PREVIEW=m.COMMENT       (sql92语法)
SELECT b.PREVIEW,b.COMMENT,m.PREVIEW,m.COMMENT FROM blogbbs b INNER JOIN blogbbs m ON b.PREVIEW=m.COMMENT  
意思为:把一张表看做很多张表进行查询,相当于对自己查询2遍

外连接介绍

外连接的查询结果为主表中的所有记录,如果从表中没有和他匹配的,则显示匹配的值,如果从表中没有和他匹配的,则显示null,外连接查询结果=内连接查询结果+主表中有而从表中没有的记录
左外连接,left join左边的世主表
右外连接,right join右边的是主表

左外连接

select * from beauty LEFT OUTER JOIN boys ON beauty.money=boys.moent; (sql99语法)

右外连接

select * from beauty RIGHT OUTER JOIN boys ON beauty.money=boys.moent; (sql99语法)

全外连接

select * from beauty FULL OUTER JOIN boys ON beauty.money=boys.moent; (sql99语法)

交叉连接

select * from beauty CROSS JOIN boys; (sql99语法)

子查询的分类

分类
    按子查询出现的位置:
        select后面:
            仅仅支持标量子查询
        form后面:
            支持表子查询
        where或having后面:
            标量子查询
            列子查询
            行子查询
        exists后面(相关子查询)
            表子查询
    按结果集的行数不同:
        标量子查询(结果集只有一行一个)
        列子查询(结果集只有一列多行)
        行子查询(结果集有一行多列)
        表子查询(结果集一般为多行多列)
    结果集就是括号里面的sql返回的结果集

标量子查询（where后面）

(结果集只有一行一个)
SELECT * FROM love WHERE name=(SELECT NAME FROM mm_doc WHERE id=130)

列子查询（where后面）

(结果集只有一列多行)
                        使用多行比较符作符
                    操作符                含义
                    IN/NOT IN         等于列表中的任意一个
                    ANY|SOME        和子查询返回的某一个值比较
                    ALL                和子查询返回的所有值比较
SELECT * FROM love WHERE name in(SELECT NAME FROM mm_doc WHERE id IN(130,131))

行子查询（where后面）

SELECT * FROM love WHERE (id,name)=(SELECT MAX(id),'开具' AS name FROM love)
//将多个字段对应

标量子查询(select后面)

SELECT love.*,(SELECT COUNT(*) FROM love) FROM love

表子查询(form后面)

SELECT * FROM (SELECT * FROM mm_doc WHERE ID=130) as mm_doc WHERE NAME="纯乐扣乐扣"
一定要给表起别名

相关子查询(exists后面)

SELECT EXISTS(SELECT * FROM mm_doc)
EXISTS：是一个boolean类型如果反回1则代表有数据,如果反回0则代表没有数据
如果EXISTS为真那么外层语句将会被执行,如果为假那么将不会执行外层语句
select username from employee where exists(select * from department where id=1)//如果子句查出来的有结果(也就是为真)那么才会执行外层语句,否则就不会执行
NOT EXISTS和上面正好相反,如果为假那么就会执行

1.REGEXP‘匹配方式’
^ 匹配字符开始的部分
$ 匹配字符结尾的部分

+ 代表1个或者多个其前的字符
String{N} 字符串出现N次
字符串{M,N} 字符串至少出现M次,最多N次 
SELECT * FROM lvshihao WHERE username REGEXP "^Q...";
