1.Shell概述

​ 大数据程序员为什么要学习shell呢？

​ 1）需要看懂运维人员编写的shell程序

​ 2）偶然编写一些简单的shell程序来管理集群，提高效率开发

​ shell是一个命令解释器,他接受应用程序/用户命令，然后操作系统内核

​ shell还是一个功能强大的编写语言,容易编写，容易调试，灵活性强

2.Shell解析器

1)Liunx提供的Shell解析器有：

cat /etc/shells

2)Centos默认的解析器是bash

echo $SHELL

3.Shell脚本入门

1.脚本格式

脚本以#!/bin/bash开头(指定解析器)

2.第一个Shell脚本:helloworld

(1)需求:创建一个shell脚本,输出helloworld

(2)案例实操：

touch hellowrld.sh创建一个sh文件
vi hellowrld.sh编辑sh文件
在hellowrld.sh中输入如下内容
#!/bin/bash
echo "lvshihao hellowrld"

(3)脚本的常用执行方式

第一种:采用bash或sh+脚本的相对路径或绝对路径(不用赋予脚本+x权限)

sh+脚本的相对路径

sh hellowrld.sh
输出 lvshihao hellowrld

sh+脚本的绝对路径

sh /lib/lvshihao/lvshihao.sh
输出 lvshihao hellowrld

bash+脚本的相对路径

bash hellowrld.sh
输出 lvshihao hellowrld

bash+脚本的绝对路径

bash /lib/lvshihao/lvshihao.sh
输出 lvshihao hellowrld

第二种：采用输入脚本的绝对路径或相对路径执行脚本(必须具有可执行权限+x)

(a)首先要赋予lvshihao.sh脚本的+x权限

chmod 777 lvshihao.sh

(b)执行脚本

相对路径

./lvshihao.sh 
输出 lvshihao hellowrld

绝对路径

/lib/lvshihao/lvshihao.sh

3.第二个shell脚本:多命令处理

(1)需求:

在/home/atguigu/目录下创建一个lvshihao.txt,在lvshihao.txt文件中增加”I love cls”

(2)实例操作

创建一个lvshihao.sh文件，文件中写入以下代码

#!/bin/bash

cd /home/atguigu  进入这个目录下

echo "I love lvshihao" >> lvshihao.txt  给lvshihao.txt文件添加"I love lvshihao"文字

4.Shell中的变量

系统变量

​ 1.常用系统变量

​ $HOME ， $PWD，$SHELL ，$ USER

​ 2.案例实操

​ (1)查看系统变量的值

[root@localhost usr]# echo $USER
root
[root@localhost usr]# echo $PWD
/usr
[root@localhost usr]# echo $HOME
/root
[root@localhost usr]# echo $SHELL
/bin/bash

​ (2)显示当前Shell中所有变量:set

[root@localhost usr]# set
BASH=/bin/bash
BASHOPTS=checkwinsize:cmdhist:expand_aliases:extquote:force_fignore:hostcomplete:interactive_comments:login_shell:progcomp:promptvars:sourcepath
BASH_ALIASES=()
BASH_ARGC=()

自定义变量

​ 1.基本语法

​ (1)定义变量:变量=值 (注意等号两边没有空格)

[root@localhost usr]# A=lvshiaho
[root@localhost usr]# echo lvshihao
lvshihao

​ (2)撤销变量:unset 变量 (也就是清除了这个变量)

[root@localhost usr]# unset A
[root@localhost usr]# echo $A

​ (3)声明静态变量:readonly变量,注意:不能unset

[root@localhost usr]# readonly B=lvshiaho
[root@localhost usr]# echo $B
lvshiaho

只有在虚拟机重启的时候才会清除这个变量

​ (4)将变量提升为全局变量,可供其他Shell程序使用(意思就是你写的shell脚本是不能用这些变量的,必须得是全局变量)

//提升全局变量之前
[root@localhost usr]# ./lvshihao.sh 

//提升全局变量之后
[root@localhost usr]# export A
[root@localhost usr]# ./lvshihao.sh 
lvshihao

​ 2.变量定义规则

​ (1)变量名称可以由字母.数字和下划线组成,但是不能以数字开头,环境变量名建议大写。

​ (2)在bash中，变量默认类型都是字符串类型,无法直接进行数值运算。

​ (3)变量的值如果有空格,需要使用双引号或单引号括起来。

特殊变量:$n

1.基本语法

$n(功能描述:n为数字,$0代表该脚本的名称,$1-$9代表第一到第九个参数,十以上的参数,十以上的参数需要用大括号包含,如${10})

2.案例实操

(1)输出该脚本文件的名称,输入参数1和输入参数2的值

#!/bin/bash

echo "$0 $1 $2 $3"

执行的结果（第一个为文件名 【1,2,3】为参数）

[root@localhost usr]# ./lvshihao.sh  1 2 3
./lvshihao.sh 1 2 3

特殊变量:$

1.基本语法

$#(功能描述:获取所有输入参数个数,常用于循环)

2.案例实操

(1)获取输入参数的个数

#!/bin/bash

echo $#

执行结果（输出了5,说明传进去5个参数）

[root@localhost usr]# ./lvshihao.sh 1 2 3 4 5

特殊变量:$*

1.基本语法

$*(功能描述:这个变量代表命令行中所有的参数,$*把所有的参数看成一个整体)

2.案例实操

#!/bin/bash

echo $*

执行结果

[root@localhost usr]# ./lvshihao.sh 1 2 3
1 2 3

特殊变量:$@

1.基本语法

$@(功能描述:这个变量代表命令行中所有的参数，不过$@把每个参数区分对待)

2.案例实操

#!/bin/bash

echo $@

执行结果

[root@localhost usr]# ./lvshihao.sh 1 2 3
1 2 3

特殊变量:$?

1.基本用法

$?（功能描述:最后一次执行的命令返回状态,如果这个变量的值为0,证明上一个命令正确执行:如果这个变量的值非0(具体是哪个数,由命令自己决定),则证明上一个命令执行不正确了.）

2.实例操作

#!/bin/bash

echo $？

执行结果

[root@localhost usr]# ./lvshihao.sh
0

6.条件判断

基本语法

[ condition ]（注意conition前后要有空格）

注意:条件非空即为true,[ atguigu ]返回true，[ ]返回false

常用判断条件

(1)两个整数之间比较

=字符串比较 -lt小于(less than)  -le小于等于(less equal)  -eq等于(equal)  -gt大于(greater than)
-ge大于等于(greater equal) -ne不等于(Not equal)

(2)按照文件权限进行判断

-r有读的权限(read) -w有写的权限(write) -x有执行的权限(execute)

(3)按照文件类型进行判断

-f文件存在并且是一个常规的文件(file) -d文件存在并是一个目录(directory) -e文件存在(existence)

实例操作

(1)23是否大于等于22

[root@localhost lvshihao]# [ 23 -ge 22 ]
[root@localhost lvshihao]# echo $?
0
[root@localhost lvshihao]# [ 23 -eq 22 ]
[root@localhost lvshihao]# echo $?
1
由此可见执行完成条件判断是不反回结果的,可以通过echo $?来查看上一个命令是否正确,来判断是否成功

(2)判断lvshihao.sh文件是否有写权限

[root@localhost lvshihao]# [ -w lvshihao.sh ]
[root@localhost lvshihao]# echo $?
0

(3)/home/atguigu/cls.txt目录中文件是否存在

[root@localhost lvshihao]# [ -e /home/atguigu/cls.txt ]
[root@localhost lvshihao]# echo $?
1
由此可见文件不存在

(4)多条件判断(&&表示前一条命令执行成功时,才执行后一条命令,|| 表示上一条命令执行失败后,才执行下一条命令)

[root@localhost lvshihao]# [ condition ] && echo ok || echo notok
ok
[root@localhost lvshihao]# [ condition ] && [  ] || echo notok 
notok

7.流程控制(重点)

if判断

基本语法

if[ 条件判断式 ];then
    程序
fi
或者
if[ 条件判断式 ]
 then
    程序
fi
注意事项:
(1)[ 条件判断式 ],中括号和条件判断式之间必须有空格
(2)if后要有空格

实例操作

(1)输入一个数字,如果是1,则输出lvshihao,如果是2,则输出521,如果是其他,什么也不出输出

#!/bin/bash

if [ $1 -eq 1 ]
then
        echo "lvshihao"
elif [ $1 -eq 2 ]
then
        echo "521"
fi

case语句

基本语法

case $变量名 in
   "值1")
           如果变量的值等于1,则执行的程序1
      ;;
   "值2")
           如果变量的值等于2,则执行的程序2
      ;;
      ...节省其他分支...
      *）
      如果变量的值都不是以上的值,则执行此程序
      ;;
 esac 
 注意事项:
 1) case行尾必须为单词 “in”,每一个模式匹配必须以右括号“)”技术。
 2）双引号“;;”表示命令序列结束,相当于java中的break
 3）最后的“*)” 表示默认模式,相当于java中的default
 4）esac代表结束

实例操作

输入一个数字,如果是1,则输出lvshihao,如果是2,则输出521,如果是其他,输出123

case $2 in
1)
        echo "lvshihao"
;;
2)
        echo "521"
;;
*)
        echo "123"
;;
esac

for循环

基本语法1

for((初始值;循环控制条件;变量变化))
    do
        循环
    done

案例实操

(1)从1加到100

#!/bin/bash

s=0
for((i=1;i<=100;i++))
do
        s=$[$s+$i]
done

基本语法2

for 变量 in 值1 值2 值3...
    do
        程序
    done

案例实操

(1)打印所有输入参数

$*执行

#!/bin/bash

for i in "$*"
do
        echo "lvshihao love $i"
done

$@执行

#!/bin/bash

for i in "$@"
do
        echo "lvshihao love $i"
done

效果是不一样的如果都是字符串的话$*代表把所有参数当做一个整体,$@则是区分对待.

while循环

基本语法

while [ 条件判断 ]
    do
        程序
    done

案例实操

(1)从1到100

#!/bin/bash

s=0

do
        s=$[$s + $i]

10.Shell工具

cut

cut的工作就是"剪",具体的就是在文件中负责剪切数据用的,cut命令从文件的每一行剪切字节,字符和字段并将这些字节,字符和字段输出。

基本用法

cut [选项参数] filename
说明:默认分隔符是制表符

选项参数说明

选项参数 功能
-f 列号,提取第几列
-d 分隔符,按照指定分隔符分割列

案例实操

(0)数据准备

[root@localhost lvshihao]# touch cut.txt
[root@localhost lvshihao]# vi cut.txt 
lvshihao lv
lvshihao shi
aini ai
sini si

(1)切割cut.txt第一列

[root@localhost lvshihao]# cut -d " " -f 1 cut.txt
lvshihao
lvshihao
aini
sini

(2)切割cut.txt第二,第三列

[root@localhost lvshihao]# cut -d " " -f 2,3 cut.txt
lv
shi
ai
si

(3)在 cut.txt文件中切割出lvshihao

[root@localhost lvshihao]# cat cut.txt | grep "lvshihao" | cut -d " " -f 1 
lvshihao
lvshihao

(4)选取系统PATH变量值,第2个”: “ 开始后的所有路径:

[root@localhost lvshihao]# echo $PATH | cut -d : -f 3-
/sbin:/opt/rh/rh-ruby23/root/usr/local/bin:/opt/rh/rh-ruby23/root/usr/bin:/usr/local/sbin:/usr/local/bin:/sbin:/bin:/usr/sbin:/usr/bin:/root/bin

(5)切割ifconfig后打印的IP地址

[root@localhost lvshihao]# ifconfig eth0 | grep "inet addr" | cut -d : -f 2 | cut -d " " -f 1

sed

sed是一种流编译器,它一次处理一行内容。处理时,把当前处理的行储蓄在临时缓冲区中,称为"模式空间",接着用sed命令处理缓冲区中的内容,处理完成后,把缓冲区的内容送往屏幕。接着处理下一行,这样不断重复,直到文件末尾。文件内容并没有改变,除非你使用重定向储蓄输出。

基本用法

sed [选项参数] ‘command’ filename

选项参数说明

选项参数 功能
-e 直接在指令列模式上进行sed的动作编辑。

命令功能描述

命令 功能描述
a 新增,a的后面可以接字符串,在下一行出现
d 删除
s 查找并替换

案例实操

(0)数据准备

[root@localhost usr]# touch sed.txt
[root@localhost usr]# vi sed.txt 
ong shen
guan zhen
wo wo
lai lai

le le

(1)将”mei nv”这个单词插入到sed.txt第二行下,打印。

[root@localhost usr]# sed '2a mei nv' sed.txt
dong shen
guan zhen
mei nv
wo wo
lai lai

le le

注意：文件并没有改变

(2)删除sed.txt文件所有包含wo的行

[root@localhost usr]# sed "/wo/d" sed.txt 
dong shen
guan zhen
lai lai

le le

(3)将sed.txt文件中wo替换为ni

[root@localhost usr]# sed "s/wo/ni/g" sed.txt 
dong shen
guan zhen
ni ni
lai lai

le le

注意:’g’表示global,全部替换

(4)将sed.txt文件中的第二行删除并将wo替换为ni

[root@localhost usr]# sed -e '2d' -e 's/wo/ni/g' sed.txt
dong shen
ni ni
lai lai

le le

awk

一个强大的文本分析工具,把文件逐行的读入,以空格为默认分隔符将每行切片,切开的部分再进行分析处理。

基本用法

awk[选项参数]  “pattern1{action1}  pattern2{action2}....”  filename 
pattern: 表示AWK在数据中查找的内容,就是匹配模式
action:在找到匹配内容时所执行的一系列命令

选项参数说明

选项参数 功能
-F 指定输入文件分隔符
-v 赋值一个用户定义变量

案例实操

(0)数据准备

[root@localhost usr]# sudo cp /etc/passwd ./

(1)搜索passwd文件以root关键字开头的所有行,并输出该行的第7列。

[root@localhost usr]# awk -F : '/^root/{print $7}' passwd
/bin/bash

(2)搜索passwd文件以root关键字开头的所以行,并输出该行第1列和第7列,中间以”,”号分割。

[root@localhost usr]# awk -F : '/^root/{print $1","$7}' passwd
root,/bin/bash

注意:只有匹配了pattern的行才会执行action

(3)只显示/etc/passwd的第一列和第七列,以逗号分割,且在所有行前面添加列名user,shell在最后一行添加”dahaige, /binzuishuai”。

[root@localhost usr]# awk -F : 'BEGIN{print "user,shell"} {print $1","$7} END{print "dahaige, bin/zuishuai"} ' passwd

注意:BEGIN在所有数据取行之前执行;END在所有数据执行之后执行。

(4)将passwd文件中的用户id增加数值1并输出

awk -F : -v i=1 '{print $3+i}' passwd

awk的内置变量

变量 说明
FILENAME 文件名
NR 已读的记录数
NF 浏览记录的域的个数(切割后,列的个数)

案例实操

(1)统计passwd文件名,每行的行号,每行的列数

[root@localhost usr]# awk -F : '{print "filename:" FILENAME ", linenumber:" NR ",columns:" NF}' passwd

(2)切割ip

[root@localhost usr]# ifconfig eth0| grep "inet addr" |awk -F : '{print $2}' |awk -F " " '{print $1}'
192.168.159.128

(3)查找sed.txt中空行所在行号

[root@localhost usr]# awk '/^$/{print NR}' sed.txt
5

sort

sort命令是在Linux里非常有用,他将文件进行排序,并将排序结果标准输出。

基本语法

sort(选项)(参数)
选项 说明
-n 依照数值的大小排序
-r 以相反的顺序来排序
-t 设置排序时所用的分隔字符。
-k 指定需要排序的列

参数：指定待排序的文件列表

案例实操

(0)数据准备

[root@localhost usr]# touch sort.sh

xz:50:2.3

[root@localhost usr]# sort -t : -nrk 3 sort.sh

cls:10:3.5
xz:50:2.3
ss:30:1.6

问题1：Shell脚本里如何检查一个文件是否存在?如果不存在该如何处理?

#!/bin/bash

if [ -f file.txt ]; then
    echo "文件存在!"
else
